Data Deletion Policy

ABX — Diagnostic & Control Android Application
Last Updated: July 14, 2026

Because ABX does not collect, upload, or store any personal data on remote servers, there is no remote data to request deletion for. You are in full control of all application data on your local device.

1. User Data Ownership & Local Control

The ABX application (the "Application") operates entirely under a local-only data model. We do not maintain user databases, cloud backups, or remote storage infrastructures. Any diagnostic configurations, log trails, or local preferences you configure exist solely within your device's private sandbox storage.

2. How to Delete All Local Data

You can instantly and completely delete all settings, databases, and local storage cached by the Application at any time. To purge all application files, perform the following standard Android system steps:

  1. Open the Settings app on your Android device.
  2. Navigate to Apps or Application Manager.
  3. Select ABX from the list of installed applications.
  4. Tap on Storage or Storage & Cache.
  5. Tap Clear Data (or Clear Storage) and confirm the action. This deletes all private database instances and configuration files.
  6. Alternatively, simply Uninstall the Application from your device, which automatically instructs the Android OS to securely wipe all data directories sandboxed for the Application.

3. No Remote Accounts or Files

Because there are no user accounts, profile registration, or cloud servers linked to ABX, you do not need to request remote deletion or data extraction from us. We do not hold, possess, or have access to any of your local settings or system logs.
