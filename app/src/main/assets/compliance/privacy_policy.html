Privacy Policy

Last updated: July 18, 2026

This Privacy Policy describes how we handle information in connection with the ABX Server application. We are committed to protecting user privacy and maintaining transparency.

1. Data Collection

Offline-First Principle: This application operates fully offline on your physical device. We do NOT collect, harvest, transmit, or store any personal data, usage statistics, user-identifiable information, or hardware tracking metrics. All sensitive credentials, device fingerprints, and local audit logs are kept strictly in localized application memory or inside isolated system databases.

2. Key Materials & Security Enclaves

All cryptographic keys (such as the EC-256 device credential keypair) are isolated securely using Android's hardware-backed Keystore system. The private key material is non-exportable, locked securely in the device's physical Hardware Security Module (HSM) / Trusted Execution Environment (TEE), and cannot be accessed or transferred off-device.

3. Third-Party Sharing

Because we do not collect any information, we do not—and cannot—share, sell, rent, or distribute any user data to third-party companies, advertising networks, or analytics providers.

4. App Permissions

To enable offline security monitoring, the application might request localized platform permissions such as:

  • POST_NOTIFICATIONS: Required to keep active background tunnels visible to the user via status indicators.
  • LOCAL_STORAGE / SANDBOX READ-WRITE: Scoped exclusively to designated folders to enforce local security validation policy guidelines.

5. Compliance

This policy complies fully with standard Google Play developer policies. The policy is accessible directly without requiring user login or registration.
